# Dual Ace: Sky Rift 개발 문서

## 1. 프로젝트 개요

`Dual Ace: Sky Rift`는 Unreal Engine C++만으로 구현한 3D 플라이트 슈팅 게임이다.  
플레이어는 두 종류의 전투기 중 하나를 선택하고, 비행 코리더 안에서 Wave 단위로 등장하는 적 드론과 최종 보스를 격추해야 한다.

핵심 목표는 다음과 같다.

- C++ 기반 게임 로직 구현
- 로비, 대시보드, 캐릭터 선택, 게임플레이, 결과 화면 구성
- 1인칭/3인칭 시점 전환
- 캐릭터별 능력 차이
- Wave 기반 난이도 상승
- HP, 데미지, 이동 속도 레벨디자인
- 로컬 리더보드 저장

## 2. 화면 흐름

```text
Lobby
  ├─ Game Play -> Character Select
  ├─ Dashboard -> Leaderboard 확인
  └─ Exit -> 게임 종료

Character Select
  ├─ Falcon 선택
  ├─ Titan 선택
  ├─ Launch -> Gameplay 시작
  └─ Back -> Lobby

Gameplay
  ├─ Wave 전투
  ├─ 보스전
  └─ Game Over / Mission Complete

Result
  ├─ 최종 점수 표시
  ├─ 도달 Wave 표시
  ├─ 리더보드 Top 5 표시
  └─ Restart
```

## 3. 조작 방식

| 입력 | 기능 |
| --- | --- |
| `A` | 왼쪽 이동 |
| `D` | 오른쪽 이동 |
| `W` | 앞으로 이동 |
| `S` | 뒤로 이동 |
| `Q` | 왼쪽으로 기울기 |
| `E` | 오른쪽으로 기울기 |
| `Left Mouse Button` | 사격 |
| `V` | 1인칭 / 3인칭 시점 전환 |
| `1` | Falcon 선택 |
| `2` | Titan 선택 |
| `Enter` | 선택 확정 / Launch |
| `R` | 게임 종료 후 재시작 |

기존 `Q/E` 속도 조절은 제거하고, 기체의 기울기 조작으로 변경했다.  
`A/D`는 좌우 이동만 수행하며 회전을 주지 않는다. 기체 회전은 `Q/E` 수동 기울기 입력에서만 발생한다.  
기존 상하 이동은 제거하고, `W/S`로 전후 이동을 수행한다.

## 4. 플레이어 기체

### 4.1 Falcon

Falcon은 빠른 이동과 빠른 연사가 장점인 경량 전투기다.

| 항목 | 값 |
| --- | --- |
| HP | 90 |
| 전진 속도 | 640 |
| 좌우 이동 속도 | 1080 |
| 상하 이동 속도 | 900 |
| 탄환 데미지 | 16 |
| 발사 간격 | 0.08초 |
| 벽 충돌 데미지 | 14 |

특징:

- 빠르게 회피 가능
- 연사력이 높아 지속 화력이 좋음
- HP가 낮고 벽 충돌 데미지가 커서 조작 실수에 취약함

### 4.2 Titan

Titan은 높은 체력과 강한 공격력을 가진 중장갑 전투기다.

| 항목 | 값 |
| --- | --- |
| HP | 170 |
| 전진 속도 | 500 |
| 좌우 이동 속도 | 760 |
| 상하 이동 속도 | 650 |
| 탄환 데미지 | 34 |
| 발사 간격 | 0.18초 |
| 벽 충돌 데미지 | 9 |

특징:

- 생존력이 높음
- 한 발 데미지가 강함
- 이동과 연사가 느려 회피가 어려움

## 5. 카메라 시스템

플레이어는 `V` 키로 1인칭과 3인칭 시점을 전환할 수 있다.

### 5.1 1인칭 시점

- 기체 전방 콕핏 위치에 카메라 배치
- 조준과 몰입감에 유리
- 시야가 좁아 회피 난이도 상승

### 5.2 3인칭 시점

- 플레이어 뒤쪽과 위쪽에서 추적
- 적 위치와 벽 경계를 확인하기 쉬움
- Wave 전투와 보스전에서 상황 파악에 유리

플레이 시작 시 `PlayerController`의 ViewTarget을 플레이어 Pawn으로 강제 지정하여 카메라가 플레이어를 따라가도록 처리했다.

## 6. 전장 공간

전장은 `SpaceArena` C++ 액터로 구성된다.

구성 요소:

- 바닥 그리드
- 좌우 경계 레일
- 노란 게이트 프레임
- 별/부유물 기준점
- 앞으로 이동할 때 뒤쪽 오브젝트를 앞쪽으로 재배치하는 반복 구조

이 구조를 통해 플레이어가 빈 공간에서 싸우는 느낌을 줄이고, 비행 코리더 안에서 전투한다는 공간감을 제공한다.

## 7. 벽 충돌 시스템

플레이어는 전장 경계를 벗어나려고 하면 위치가 제한되고 데미지를 받는다.

경계값:

| 축 | 최소 | 최대 |
| --- | --- | --- |
| Y | -920 | 920 |
| Z | 120 | 920 |

벽 충돌 데미지:

| 기체 | 데미지 |
| --- | --- |
| Falcon | 14 |
| Titan | 9 |

데미지는 즉시 반복 적용되지 않도록 `0.45초` 쿨다운을 둔다.

## 8. 적 종류

### 8.1 기본 드론

- 가장 기본적인 적
- 평균적인 체력, 속도, 데미지
- Wave 1부터 등장
- 플레이어 위치를 추적하여 충돌을 시도함

### 8.2 돌격 드론

- 빠른 이동 속도
- 낮은 체력
- 플레이어에게 빠르게 접근해 충돌 위험을 높임
- Wave 3부터 등장
- 플레이어 추적 속도가 빠름

### 8.3 방어 드론

- 높은 체력
- 느린 이동 속도
- 높은 접촉 데미지
- Wave 4부터 등장
- 느리지만 버티면서 플레이어에게 접근함

### 8.4 Command Carrier 보스

- 최종 Wave에서 등장
- 높은 HP
- 큰 충돌 범위
- 플레이어를 추적하며 약한 흔들림 패턴을 섞어 접근

보스 스탯:

| 항목 | 값 |
| --- | --- |
| HP | 760 |
| 이동 속도 | 105 |
| 접촉 데미지 | 55 |
| 처치 점수 | 2500 |

## 9. HP 시스템

HP는 플레이어, 몬스터, 보스 모두에게 적용된다.

### 9.1 플레이어 HP

- HUD 왼쪽 상단에 HP 바와 숫자로 표시
- HP가 0 이하가 되면 게임 실패 처리

### 9.2 몬스터 HP

- 적 머리 위에 월드 HP 바 표시
- 남은 HP 숫자 표시
- HP가 0 이하가 되면 점수 획득 후 파괴

### 9.3 보스 HP

- 화면 상단 중앙에 보스 전용 HP 바 표시
- 보스 머리 위에도 HP 바 표시
- 보스를 처치하면 Mission Complete 처리

## 11. 이동 속도 레벨디자인

### 11.1 플레이어

| 기체 | 전진 속도 | 좌우 속도 | 상하 속도 |
| --- | --- | --- | --- |
| Falcon | 640 | 1080 | 900 |
| Titan | 500 | 760 | 650 |

### 11.2 적

적 이동 속도는 Wave가 올라갈수록 증가한다.  
돌격 드론은 기본 속도의 `1.42배`, 방어 드론은 기본 속도의 `0.72배`로 설정했다.

## 12. Wave 레벨디자인

| Wave | 스폰 간격 | 동시 스폰 수 | 기본 적 속도 | 기본 적 HP | 기본 적 데미지 | 특징 |
| --- | --- | --- | --- | --- | --- | --- |
| 1 | 1.45초 | 1 | 560 | 35 | 14 | 기본 드론만 등장 |
| 2 | 1.20초 | 1 | 640 | 45 | 18 | 기본 드론 강화 |
| 3 | 1.00초 | 2 | 720 | 52 | 20 | 돌격 드론 추가 |
| 4 | 0.82초 | 2 | 790 | 62 | 24 | 방어 드론 추가 |
| 5 | 0.68초 | 3 | 860 | 72 | 28 | 적 조합 강화 |
| 6 | 보스전 | - | - | - | - | Command Carrier 등장 |

Wave가 올라갈수록 다음 요소가 강화된다.

- 스폰 간격 감소
- 동시 스폰 수 증가
- 적 HP 증가
- 적 이동 속도 증가
- 적 접촉 데미지 증가
- 특수 적 등장 확률 증가

## 16. 최신 입력 및 추적 수정

- 게임 시작 직후 UI 포커스 때문에 `W/A/D` 입력이 씹히지 않도록 `PlayerController`가 플레이 중 `W/A/S/D/Q/E` 키 상태를 매 프레임 직접 동기화한다.
- `W`는 전진, `S`는 후진, `A/D`는 좌우 이동만 담당하며, `A/D` 이동 중 자동 회전은 발생하지 않는다.
- 일반 적과 보스는 플레이어와 같은 Z 높이에서 추적한다.
- 적 스폰 위치도 플레이어의 현재 Z 높이에 맞춰 생성된다.
